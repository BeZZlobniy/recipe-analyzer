Тема: Правила по насыщенным жирам и натрию

Назначение для MVP:
Файл нужен для отдельного rule-based блока предупреждений даже тогда, когда RAG не нашел много текстового контекста.

Правила по насыщенным жирам:
- если есть сливочное масло + сливки + сыр -> saturated_fat_risk = high
- если есть жирное красное мясо или переработанное мясо -> saturated_fat_risk повышается
- если основное масло рецепта оливковое/растительное и нет жирных молочных продуктов -> saturated_fat_risk понижается
- фритюр и панировка могут усиливать общий неблагоприятный профиль блюда

Правила по натрию:
- если есть соевый соус, бульонный кубик, колбаса, бекон, соленый сыр, маринад или консервированный суп -> sodium_risk повышается
- если несколько таких ингредиентов сочетаются в одном рецепте -> sodium_risk = high
- если есть пометка low sodium / reduced sodium -> sodium_risk можно понизить на один уровень
- для неявных случаев учитывать cooking notes и готовые соусы

Приоритеты предупреждений:
- allergen warning > strict profile incompatibility > sodium/saturated fat risk > general healthy eating note

Шаблоны интерпретации:
- «В рецепте несколько источников насыщенных жиров, поэтому блюдо может быть менее подходящим при контроле липидного профиля»
- «В рецепте присутствуют несколько выраженных источников натрия, поэтому блюдо может быть менее подходящим для low-sodium профиля»

Источниковая опора:
- American Heart Association: sodium guidance
- NHLBI: TLC / DASH principles
